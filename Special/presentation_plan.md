Компании обучают сотрудников через PDF-рассылки, Word-тесты и электронную почту — нет единой точки контроля, нет автоматической проверки, нет аналитики прогресса.

**Статистика / аргументы (подобрать 1–2 под свой вуз):**
- Рынок корпоративного e-learning растёт ежегодно на 15–20% (данные Research and Markets, 2024).
- По данным LinkedIn Workplace Learning Report: 94% сотрудников готовы остаться в компании дольше, если та инвестирует в их обучение.
- Ручная организация обучения даёт ошибки фиксации результатов в 20–30% случаев (нет автоматической проверки).

## СЛАЙД 4 — Предметная область (часть 1)

**Участники системы:**

| Роль | Права и обязанности |
|---|---|
| Администратор | Создаёт учётные записи, назначает преподавателей, управляет курсами и правами |
| Преподаватель | Добавляет/редактирует модули, уроки, задания; принимает заявки на пересдачу; просматривает результаты |
| Обучающийся | Проходит курсы, выполняет задания, накапливает XP, просматривает прогресс и достижения |
| Эксперт | Проверяет задания с развёрнутым ответом и файлами; выставляет баллы по рубрикатору |

**Ключевые объекты:**
Курс → Модуль → Урок → Материалы (видео/PDF) + Тест (MultipleChoice / ShortAnswer)

**Базовые процессы:**
1. Регистрация и назначение ролей администратором
2. Создание учебного контента преподавателем
3. Прохождение материалов и тестов обучающимся
4. Проверка заданий и выдача обратной связи

## СЛАЙД 6 — Проблема предметной области

**Конкретные тезисы:**

- **Отсутствие единой платформы** → данные о прогрессе сотрудников разбросаны по почте, Excel, мессенджерам.
- **Невозможность автоматической проверки** → преподаватель тратит N часов на ручную проверку тестов.
- **Нет отслеживания прогресса в реальном времени** → руководитель получает отчёт только по запросу и с задержкой.
- **Человеческий фактор** → ошибки при ручном вводе результатов, потеря данных.
- **Низкая мотивация обучающихся** → нет геймификации, нет обратной связи о прогрессе.

**Вывод (одна фраза):**
«Существующий процесс корпоративного обучения не масштабируется и не даёт прозрачности — требуется специализированная информационная система».

## СЛАЙД 7 — Постановка задачи / Требования к системе

**Цель ИСО:**
Автоматизировать цикл корпоративного обучения: от назначения курса до получения результата, с прозрачным отслеживанием прогресса.

**Функциональные требования:**

| # | Требование | Реализовано |
|---|---|---|
| Ф1 | Ролевая авторизация (Admin, Teacher, Student, Expert) | ✅ ASP.NET Identity |
| Ф2 | Иерархия контента: Курс → Модуль → Урок → Материалы/Тест | ✅ XML-контент |
| Ф3 | Материалы: YouTube-видео, PDF-документы | ✅ |
| Ф4 | Тесты: выбор ответа, короткий ответ | ✅ (ShortAnswer — AI в планах) |
| Ф5 | XP-система: 10/25/45 XP за урок по уровню сложности | ✅ |
| Ф6 | Отслеживание прогресса (материалы + тест) | ✅ |
| Ф7 | Достижения (бейджи по прогрессу курса: 33%/66%/100%) | ✅ |
| Ф8 | Мини-игры (Unity WebGL, по 1 игре на модуль) | ✅ (в разработке) |
| Ф9 | Мультиязычность (RU/EN) | ✅ XML локализация |
| Ф10 | Заявка на пересдачу теста с комментарием | 🔄 В планах |

**Нефункциональные требования:**
- Время отклика страницы < 1 секунды
- Поддержка мобильных устройств (адаптивная вёрстка)
- Тёмная и светлая тема интерфейса

## СЛАЙД 9 — Описание ИС (Архитектура + Скриншоты)

### IDEF0 TO-BE (как будет / как есть в Tria)

```
Компания → [Tria: Администратор создаёт учётки] → Обучающийся видит свои курсы
         → [Обучающийся проходит урок + тест] → Система автоматически считает XP
         → [Прогресс сохраняется в БД] → Преподаватель видит аналитику
         → [Система выдаёт достижения]
```

### Техническая архитектура

```
Браузер (Клиент)
       ↕  HTTPS
ASP.NET Core 10.0 (Razor Pages, C#)
  ├── Pages/ (Index, Dashboard, CourseDetail, LessonDetail, TakeTest, Achievements, Progress...)
  ├── Services/ (LearningService — парсинг XML, ProgressService — XP и прогресс)
  ├── Models/ (Course, Module, Lesson, UserLessonProgress, UserTestAttempt)
  └── Data/ (ApplicationDbContext — EF Core)
       ↕
Microsoft SQL Server (пользователи, прогресс, попытки тестов)
       +
XML-файлы (курсы, модули, уроки, тесты — Content/Courses/RU/*.xml)
       +
wwwroot/ (Unity WebGL игры, PDF, статика)
```

**Стек технологий:**
- Backend: ASP.NET Core 10.0, C#, Entity Framework Core
- Frontend: Razor Pages, CSS (кастомный дизайн, тёмная/светлая тема)
- БД: Microsoft SQL Server
- Контент курсов: XML (парсится LearningService)
- Игровые модули: Unity WebGL (Brotli-сжатие)
- Аутентификация: ASP.NET Identity + Cookie
- Локализация: XML UI-файлы (RU/EN)

**Типичные вопросы комиссии:**
1. _Чем отличается от Moodle?_ → «Tria ориентирована на корпоративный сегмент, легковесна, не требует сложного администрирования. Контент хранится в XML — просто добавить новый курс без UI-интерфейса».
2. _Почему XML, а не БД для контента?_ → «Курсовой контент редко меняется, XML удобно версионировать через git и редактировать напрямую. В будущем планируется CRUD-панель».
3. _Как обеспечена безопасность данных?_ → «ASP.NET Identity, HTTPS, ролевая авторизация, хэширование паролей, ФЗ-152 — личные данные только в SQL Server».
4. _Как тестировалась система?_ → «Функциональное тестирование: каждая роль, каждая страница. Сценарное: полный цикл от регистрации до получения достижения».
